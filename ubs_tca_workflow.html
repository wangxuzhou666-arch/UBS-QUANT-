S0.1 Winsorization Outlier Treatment (P1–P99) ▼

TCA 数据具有厚尾特性（fat-tail）。将每列截断至 [P1, P99]，保留分布形态但消除极端杠杆点。HC3 稳健标准误可修正异方差，但无法处理极端杠杆点——因此 Winsorization 是前置必要步骤。

df[col] = df[col].clip(lower=p1, upper=p99)
字段P1 下限P99 上限
IS_Slippage~-283 bps~+723 bps
ordersize~0.72~362.52
averageSpread~0.05~15.01
duration~0.03~1.00
Why: 极端值（fat-finger / 黑天鹅）会严重偏移 OLS 系数，必须在回归前处理。
EDA 实证： Winsor 前 algo_2 右尾延伸至 2500 bps，截尾后收敛至约 850 bps。处理后 algo_1 峰值更高（低成本区集中），algo_2 分布更宽（成本离散度更大）。两组分布至此才具有可比性。

Q2 名义价值加权 IS Notional-Weighted IS + Pareto ▼

简单均值将 100 股订单与 10,000 股订单等权处理——不合理。名义价值加权 IS 反映真实 P&L 影响：

NW_IS = Σ(IS_i × notional_i) / Σ(notional_i)

Pareto 分析：按 notional 排序后，前 10% 的大额订单贡献绝大部分总执行成本。

Contribution_i = (IS_i × notional_i) / Σ(IS_j × notional_j) × 100%
意义： 对路由决策有直接指导——优先优化高名义价值订单。
EDA 实证： NW_IS：algo_1 = 206.9 bps，algo_2 = 278.4 bps，差距 +71.5 bps。Pareto 曲线显示前 20% 大单贡献约 70% 累积 IS。algo_2 累积曲线上升更陡峭，说明大单成本占比更高——大单路由选择是成本控制的核心战场。

Figure Q2 — Notional-Weighted IS 与 Pareto 累积贡献曲线

加权 IS 建立 → 验证订单集同质性
Q3 特征分布分析 Feature Distribution + KS Test ▼

绘制各特征在两算法间的 KDE 分布图，KS 检验判断订单是否同质。并构造新特征 exec_intensity 作为 POV 代理：

exec_intensity = ordersize / max(duration, 1e-4) # 执行强度越高 → 在更短时间内向市场注入更大量 → 冲击更大 ks_stat, ks_p = stats.ks_2samp(df_algo1[feat], df_algo2[feat]) # p < 0.05 → 特征分布显著不同 → 混淆变量确认
关键发现： 特征分布显著不同 → 两算法被路由了不同难度的订单 → Q1 的原始比较结论不可靠，必须进行控制变量回归（Q5）。
EDA 实证（路由偏差证据）：
✅ ordersize：algo_2 log 中心约 4 vs algo_1 约 3，algo_2 被分配了更大的订单
✅ exec_intensity：algo_2 分布整体右移约 1 个单位（执行强度更高）
❌ volatility / averageSpread / duration：两组无差异，不是路由偏差来源

Q5 核心 OLS 因果回归 Core OLS — Causal Regression ▼

控制订单难度后，algo_2 相比 algo_1 的真实 IS 溢价是多少？理论驱动特征变换：

log_vol = log(volatility) # 线性化乘法关系 log_spread = log(averageSpread) # 同上 sqrt_size = sqrt(ordersize) # Almgren-Chriss 平方根冲击定律 sqrt_intensity = sqrt(exec_intensity) # POV 代理 spread_x_size = log_spread × sqrt_size # 交互项：双重惩罚 algo_dummy = 1 if algo_2 else 0 # ← 关键哑变量 IS = β₀ + β₁·log_vol + β₂·log_spread + β₃·sqrt_size + β₄·duration + β₅·sqrt_intensity + β₆·spread_x_size + β₇·algo_dummy + ε
β₇ 是最终答案：
β₇ > 0 且 p < 0.05 → algo_2 更贵（bps）
β₇ < 0 且 p < 0.05 → algo_2 更省（bps）
p ≥ 0.05 → 无显著差异，原始差距完全由路由偏差解释
EDA 实证： β₇ = -83.58 bps（p=0.0000，95% CI: [-86.50, -80.66]）
残差图呈喇叭形（异方差成立）→ 需 HC3 修正。QQ 图两端翘起（厚尾）。
系数图：log_vol 最强（约 +200 bps）> sqrt_size（约 +30 bps）> algo_dummy（-84 bps）。
结论方向反转：Q1 原始 +40 bps → Q5 控制后 -83.6 bps，路由偏差被完全识别。

Figure Q5 — OLS 诊断图（残差 / QQ / 系数与 95% CI）

OLS 结果确立 → 三层稳健性交叉验证
Q5R 稳健性检验三层 Robustness: HC3 + Huber + Quantile ▼

单一 OLS 可能受异方差或残差异常值影响，三层交叉验证：

① Breusch-Pagan + HC3 ② Huber ε=1.35 ③ Quantile Q10–Q90
# ① 异方差检验 bp_stat, bp_p, _, _ = het_breuschpagan(residuals, X) # p < 0.05 → 必须用 HC3 标准误 # ② Huber 回归 HuberRegressor(epsilon=1.35, max_iter=500) # ③ 分位数回归 for q in [0.10, 0.25, 0.50, 0.75, 0.90]: qr = smf.quantreg(formula, df).fit(q=q, max_iter=2000)
验证逻辑： Huber 与 OLS 的 algo_dummy 符号一致 → 对残差异常值稳健。分位数回归检验成本溢价是否集中在尾部（极端难订单）。
EDA 实证（分位数回归）：
Q10: -63 bps · Q25: -70 bps · Q50: -76 bps · Q75: -80 bps · Q90: -78 bps
全分位段均为负 → algo_2 优势全分布均匀分布，不是极端单撑起来的。越难的订单（高 IS 端），algo_2 优势越明显。

P&L 经济显著性转化 Economic Significance — Dollar Impact ▼

将统计显著的 β₇（单位：bps）转化为交易台实际可感知的美元损益：

Dollar Impact = (β₇ / 10,000) × Total Notional of algo_1 orders # 例：β₇ = +50 bps, Total Notional = 10 亿 CNY # Impact = (50 / 10000) × 1,000,000,000 = 500 万 CNY / year # → algo_2 每年多花 500 万 CNY
决策桥梁： 将统计结论（bps）翻译为商业语言（CNY），供交易台直接决策是否切换算法。
EDA 实证： algo_1 总名义金额 = 6,571 亿 CNY。β₇ = -83.58 bps。
Dollar Impact = (-83.58 / 10,000) × 657,140,627,123 = -54.9 亿 CNY / year（节省）
每交易 100 万 CNY 节省 8.36 元，直接体现在执行成本报告中。
建议：立即启动算法迁移评估。

Q6 部分成交与存活偏差 Partial Fills & Survivorship Bias ▼

题目假设全量成交。Q6 放开此假设——揭示"作弊算法"的存在：只在市场朝有利方向移动时成交，报告 IS ≈ 0，但实际通过极低成交率换来虚假优势。

Expanded_IS_i = r_i × IS_filled_i + (1 - r_i) × OppCost_i r_i = filled_qty / order_qty # 成交率 OppCost = IS_Slippage + Normal(50, 80) # 未成交机会成本 # 作弊算法：成交率与 IS 负相关 cherry_fill = clip(1 - 0.0005 × IS_Slippage + noise, 0.05, 1.0) # 中性算法：成交率独立于 IS neutral_fill = clip(0.6 + noise, 0.05, 1.0)
结论： 报告 IS ≈ 0 ≠ 算法优秀。必须同时公示成交率 + Expanded IS，否则指标可被操纵。
EDA 实证： Cherry-picker 成交率几乎全部堆在 1.0（刻意回避高 IS 单不完全成交）。
Filled-only IS（有偏）= 223 bps · Cherry Expanded = 224 bps · Neutral Expanded = 233 bps
偏差仅 10 bps（约 4.3%），Q5 主结论（83.6 bps）不受影响，但方向明确：cherry-picking 让算法看起来比实际好。

Q7 LightGBM + SHAP 非线性验证 Non-Linear Cross-Validation ▼

OLS 假设线性关系，但市场冲击在流动性阈值处可能发生非线性跳跃。LightGBM 捕捉非线性，SHAP 提供理论严格的特征归因：

n_estimators=300 lr=0.05 num_leaves=31 subsample=0.8 early_stop=30
explainer = shap.TreeExplainer(lgb_model) shap_values = explainer.shap_values(X_test) # 验证 algo_dummy 方向是否与 OLS 一致 mean_shap_algo2 = shap_values[X_test['algo_dummy']==1, algo_idx].mean()
Shapley 公理含义
效率性所有 SHAP 值之和 = 预测值 − 基准值
对称性贡献相同的特征获得相同 SHAP
虚拟性无影响特征 SHAP = 0
可加性组合模型 SHAP = 各模型之和
最终交叉验证： OLS β₇ 符号 ↔ SHAP(algo_dummy) 方向一致 → 结论在线性与非线性框架下均稳健。
EDA 实证（SHAP 特征重要性排名）：
sqrt_size ≈ 75 · log_vol ≈ 60 · algo_dummy ≈ 30 · sqrt_intensity ≈ 12 · log_spread ≈ 2
algo_dummy SHAP：algo_2=1 的点集中在负值区（降低 IS）→ 与 OLS β₇ < 0 方向完全一致。
log_spread SHAP 几乎全在 0 附近 → spread 对 IS 贡献极微，与 Spearman ρ=0.002 吻合。
